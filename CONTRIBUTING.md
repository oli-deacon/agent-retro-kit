# Contributing

Thanks for improving the kit.

## Principles

- Keep the kit lightweight.
- Prefer process clarity over framework complexity.
- Keep templates short enough that a small team will actually use them.
- Avoid baking in assumptions about a single agent product.

## Good contributions

- clearer templates
- better example retros or scorecards
- optional collectors for specific agent platforms
- safer guidance around verification and data handling
- improvements that reduce maintenance cost

## Changes to avoid unless strongly justified

- adding heavy infrastructure requirements
- assuming access to proprietary thread APIs
- turning the retro into a full analytics platform
- collecting more fields than small teams can maintain

## Pull request guidance

- explain the problem being solved
- keep examples realistic and concise
- note whether a change is tool-agnostic or tool-specific
- if you add a tool-specific guide, keep the core templates generic
